# 테스트 시나리오

## 환경
- 호스트: Galaxy S22 (실기기)
- 게스트: Pixel 6 (에뮬레이터, 10.0.2.2로 참가)
- adb forward 필요: `adb -s R3CT60D20XE forward tcp:41235 tcp:41235 && adb -s R3CT60D20XE forward tcp:41236 tcp:41236`

## 시나리오

### 1. 기본 재생
- 호스트: 방 만들기 → 파일 선택 → 재생
- 게스트: 참가 → 동기화 완료 후 자동 재생
- **확인**: 두 기기에서 동시에 같은 위치 재생되는지

### 2. 중간 참가
- 호스트: 재생 중인 상태
- 게스트: 뒤늦게 참가
- **확인**: 게스트가 호스트의 현재 위치부터 재생되는지

### 3. 재생 제어
- 호스트: 일시정지 / 재개 / 타임라인 seek
- **확인**: 게스트에 즉시 반영되는지

### 4. 싱크 보정
- 재생 상태로 30초 이상 유지
- **확인**: 드리프트 발생 시 속도 보정 로그 출력, 차이가 줄어드는지

### 5. 게스트 퇴장 (직접)
- 게스트: 뒤로가기로 방 나가기
- **확인**: 호스트 로그에 즉시 "퇴장" 표시되는지

### 6. 호스트 퇴장
- 호스트: 방 나가기
- **확인**: 게스트에 "호스트가 방을 나갔습니다" → 자동 퇴장

### 7. 네트워크 끊김
- 게스트 또는 호스트의 WiFi 끄기
- **확인**: WiFi 끊김 감지 → 자동 퇴장 / heartbeat 타임아웃(9초)으로 퇴장 감지

### 8. 백그라운드 재생
- 재생 중 앱을 홈으로 내리기
- **확인**: 소리가 계속 나오는지, 알림바/잠금화면에 재생 컨트롤 표시되는지

### 9. 파일 변경
- 호스트: 재생 중 다른 파일 선택
- **확인**: 게스트가 새 파일로 자동 전환되는지

### 10. 볼륨/음소거
- 게스트: 볼륨 조절 및 음소거 토글
- **확인**: 게스트 로컬에서만 적용, 호스트에 영향 없는지
